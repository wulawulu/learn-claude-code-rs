# s05: Skill Loading

s04 让 agent 可以把子任务委派给子代理，从而隔离上下文。接下来要解决的是另一个问题：专业知识不应该全部塞进 system prompt。

如果把所有规范、示例、项目约定、工作流都提前写进 system prompt，agent 一开始就会背上大量无关上下文。更好的方式是：先告诉 agent 有哪些技能可用，当任务需要时，再通过工具按需加载完整内容。

本章加入 `load_skill` 工具和 `SkillRegistry`，实现一个轻量的技能加载系统。

## 运行方式

```bash
cargo run -p s05_skill_loading
```

## 本章新增能力

- 扫描 `skills/` 目录下的 `SKILL.md`。
- 解析技能 frontmatter，提取 `name` 和 `description`。
- 在 system prompt 中列出可用技能摘要。
- 提供 `load_skill` 工具，按名称加载完整技能内容。
- 使用 `Arc<SkillRegistry>` 在 agent 状态和工具之间共享技能注册表。

## 为什么需要 Skill

工具解决的是“agent 能做什么”，skill 解决的是“agent 在某类任务中应该怎么做”。

典型例子：

- code review 检查清单。
- MCP server 编写步骤。
- PDF 处理命令和库选择。
- 某个团队的代码风格。
- 某个业务领域的操作流程。

这些知识不是每次任务都需要。把它们做成 skill 后，agent 可以先看到简短描述，只有在相关任务中才加载全文。

## Skill 文件格式

一个 skill 是一个 `SKILL.md` 文件，可以带 YAML frontmatter：

```markdown
---
name: code-review
description: Perform thorough code reviews with security, performance, and maintainability analysis.
---

# Code Review Skill

...
```

frontmatter 用于生成技能摘要，正文用于真正加载到上下文。

如果没有 `name`，系统会使用 skill 所在目录名作为 fallback。

## SkillRegistry

核心结构在 [`src/skill.rs`](./src/skill.rs)：

```rust
pub struct SkillRegistry {
    skills_dir: PathBuf,
    skills: HashMap<String, SkillDocument>,
}
```

加载流程：

```text
遍历 skills_dir
  -> 找到所有 SKILL.md
  -> 读取文件
  -> 解析 frontmatter
  -> 构造 SkillDocument
  -> 按 name 放入 HashMap
```

遍历使用 `walkdir`：

```rust
for entry in WalkDir::new(&self.skills_dir)
    .into_iter()
    .filter_map(Result::ok)
    .filter(|entry| entry.file_type().is_file())
    .filter(|entry| entry.file_name().to_str() == Some("SKILL.md"))
```

这允许 skill 目录有自己的子文件、脚本和 reference 文档。当前版本只加载 `SKILL.md` 本体，后续可以扩展为按需加载引用文件。

## Frontmatter 解析

`parse_frontmatter()` 支持 LF 和 CRLF：

```rust
let text = text.replace("\r\n", "\n");
```

然后按 Markdown 常见格式拆分：

```text
---
name: ...
description: ...
---

body
```

frontmatter 使用 `serde_yaml` 反序列化：

```rust
#[derive(Debug, Default, Deserialize)]
struct SkillFrontmatter {
    name: Option<String>,
    description: Option<String>,
}
```

这一章已经给解析逻辑加了单元测试，保证不同换行风格都能工作。

## 在 System Prompt 中暴露技能摘要

agent loop 每次启动时构造 system prompt：

```rust
let system = format!(
    r#"You are a coding agent at {}.
Use load_skill when a task needs specialized instructions before you act.

Skills available:
    {}
"#,
    std::env::current_dir()?.display(),
    self.skill_registry.describe_available()
);
```

这里没有把所有 skill 正文塞进去，只列出：

```text
- agent-builder: ...
- code-review: ...
- mcp-builder: ...
- pdf: ...
```

模型可以根据用户任务判断是否需要调用 `load_skill`。

## load_skill 工具

`load_skill` 的输入很简单：

```json
{
  "name": "code-review"
}
```

实现位于 [`src/tool/load_skill.rs`](./src/tool/load_skill.rs)：

```rust
Ok(self.registry.load_full_text(name))
```

返回内容会被包装成：

```xml
<skill name="code-review">
...
</skill>
```

这样模型能清楚地区分“这是技能内容”而不是普通用户消息。

## 为什么使用 Arc

`SkillRegistry` 同时被两个地方需要：

- `LoopState`：用于生成 system prompt 中的技能摘要。
- `LoadSkillTool`：用于按名称加载完整 skill。

所以本章使用：

```rust
Arc<SkillRegistry>
```

技能注册表在启动时加载，之后只读共享。`Arc` 让多个所有者共享同一份注册表，不需要复制整个技能集合。

## Tool 与 Skill 的区别

Tool 是可执行能力：

```text
read_file, write_file, bash, edit_file
```

Skill 是可加载知识：

```text
code-review checklist, mcp server workflow, pdf processing guide
```

一个成熟的 agent harness 通常两者都需要：

- 工具让模型能改变环境。
- 技能让模型在特定领域做得更专业。

## 本章的局限

- skill 只在启动时扫描，运行中新增文件不会自动刷新。
- `load_skill` 只加载 `SKILL.md`，不会自动加载 references 或 scripts。
- 没有记录某个 skill 是否已经加载过。
- 没有 skill 权限、版本、依赖关系。
- 技能内容仍然进入普通上下文，后续需要配合上下文压缩。
