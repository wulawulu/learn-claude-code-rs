# s09: Memory System

Previous chapters keep state inside a single session. Once the process exits, important preferences, facts, and feedback are lost.

This chapter adds a memory system. The agent can save durable notes and load them into future system prompts.

## Run

```bash
cargo run -p s09_memory_system
```

## New Capabilities

- Add a `MemoryManager`.
- Store memory under `.claude/memory`.
- Add a `save_memory` tool.
- Support different memory categories.
- Load memory into the system prompt.
- Keep memory separate from ordinary conversation history.

## Why Memory

Agent context is not durable. It can be compacted, truncated, or lost between runs.

Memory is for information that should survive:

- User preferences.
- Project conventions.
- Important facts.
- Feedback.
- Useful references.

The agent should not remember everything. It should save only information that is likely to matter later.

## MemoryManager

`MemoryManager` owns the memory directory and provides operations for saving and loading memory.

Memory is stored as Markdown so it remains readable and editable by humans.

Typical categories include:

- preferences
- facts
- feedback
- references

The exact schema is intentionally lightweight in this teaching version.

## save_memory Tool

The model can call:

```json
{
  "category": "preference",
  "content": "The user prefers concise final answers."
}
```

The tool writes durable memory and returns a confirmation.

## Loading Memory

At startup, the system prompt includes memory guidance and current memory content.

This means the model can use memory without first calling a tool. Saving new memory is explicit; reading existing memory is part of prompt construction.

## Memory vs Context

Context:

- Short-term conversation history.
- Includes tool calls and tool results.
- Can be compacted.

Memory:

- Long-term durable notes.
- Should be curated.
- Loaded into future sessions.

Keeping them separate avoids turning memory into a transcript dump.

## Rust Notes

- Use a manager for durable memory operations.
- Keep memory files human-readable.
- Inject memory through system prompt construction.
- Expose saving through a tool instead of letting the model write arbitrary memory files.

## Limits

- No semantic search.
- No memory ranking.
- No duplicate detection.
- No automatic memory extraction.
- Memory files are simple Markdown.
