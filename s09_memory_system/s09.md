# s09: Memory System

前面的 `context` 只在一次运行中存在。程序退出后，对话历史、用户偏好和项目经验都会丢失。

本章加入 memory system，让 agent 可以把值得长期保留的信息写入 `.memory/`，并在之后的会话中重新加载到 system prompt。

## 运行方式

```bash
cargo run -p s09_memory_system
```

运行过程中，agent 可以调用 `save_memory` 工具保存记忆。

## 本章新增能力

- 新增 `.memory/` 目录作为持久化记忆存储。
- 新增 `save_memory` 工具。
- 支持四类 memory：
  - `user`
  - `feedback`
  - `project`
  - `reference`
- 启动时加载已有 memory。
- 构建 system prompt 时注入 memory 内容。
- 自动维护 `.memory/MEMORY.md` 索引。
- 提供明确的 memory 保存准则。

## Memory 解决什么问题

Context 是短期记忆，memory 是长期记忆。

适合保存为 memory 的信息包括：

- 用户偏好：例如“总是先跑测试”。
- 用户纠正：例如“不要改这个目录”。
- 非显然项目约定：例如“这个旧模块不能动，因为兼容客户部署”。
- 外部资源位置：例如 dashboard、ticket board、文档链接。

不适合保存的包括：

- 当前任务进度。
- 分支名、PR 编号等临时状态。
- 代码里一眼能查到的事实。
- 密钥、token、密码等敏感信息。

## Memory 文件格式

每条 memory 是一个 Markdown 文件，带 frontmatter：

```markdown
---
name: prefer_cargo_check
description: User prefers checking Rust code before summarizing
type: user
---
When editing Rust code, run `cargo check` before giving the final response when practical.
```

文件名会由 `name` 清洗后生成。索引文件是：

```text
.memory/MEMORY.md
```

索引只保存摘要，具体内容保存在各自的 `.md` 文件里。

## MemoryType

```rust
pub enum MemoryType {
    User,
    Feedback,
    Project,
    Reference,
}
```

四类 memory 的边界：

- `user`：用户偏好。
- `feedback`：用户纠正和反馈。
- `project`：非显然项目事实、约定和决策原因。
- `reference`：外部资源位置。

类型通过 `strum` 支持字符串解析和展示。

## MemoryManager

`MemoryManager` 负责：

- 扫描 `.memory/`。
- 解析 memory frontmatter。
- 加载所有 memory 到内存。
- 生成 prompt section。
- 保存新 memory。
- 重建索引。

启动时：

```rust
let memory_manager = Arc::new(Mutex::new(get_memory_manager(
    std::env::current_dir()?.join(MEMORY_DIR),
)?));
```

这里用 `Arc<Mutex<_>>` 是因为：

- `LoopState` 需要读取 memory 构建 system prompt。
- `save_memory` 工具需要修改 memory。
- 两者共享同一个 manager。

## save_memory 工具

输入 schema：

```json
{
  "name": "prefer_cargo_check",
  "description": "User prefers cargo check before final response",
  "type": "user",
  "content": "When editing Rust code, run cargo check when practical."
}
```

工具执行后会：

- 写入 `.memory/<safe_name>.md`。
- 更新内存中的 `HashMap`。
- 重建 `.memory/MEMORY.md`。

## 注入 system prompt

每轮 agent loop 都会调用：

```rust
let system = self.build_system_prompt()?;
```

`build_system_prompt()` 会把三部分拼起来：

```text
基础角色说明

已加载 memory

memory guidance
```

memory guidance 会告诉模型什么时候应该保存 memory、什么时候不应该保存，避免把临时信息或敏感信息写进长期存储。

## Rust 实现要点

- 用 Markdown + YAML frontmatter 存储人类可读的 memory。
- 用 `walkdir` 扫描 `.memory/`。
- 用 `serde_yaml` 解析 frontmatter。
- 用 `Arc<Mutex<MemoryManager>>` 在 state 和 tool 之间共享可变状态。
- 用 `sanitize_name()` 控制文件名安全。

## 本章的局限

- 没有 memory 删除和编辑工具。
- memory 没有去重或冲突解决。
- 所有 memory 都注入 prompt，数量多时会膨胀。
- 没有基于语义检索的按需 memory 加载。
- 没有权限审核，模型可以直接保存 memory。
