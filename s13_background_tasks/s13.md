# s13: Background Tasks

到目前为止，agent 调用工具时都要同步等待结果。如果命令很慢，例如长时间测试、构建、下载或数据处理，整个 agent loop 会被卡住。

本章加入后台任务机制，让 agent 可以启动一个长期运行的 shell 命令，立即拿到 task id，然后在后续轮次查询或自动接收完成通知。

## 运行方式

```bash
cargo run -p s13_background_tasks
```

运行后会使用：

```text
.runtime-tasks/
```

保存后台任务记录和输出。

## 本章新增能力

- 新增 `background_run` 工具，启动后台 shell 命令。
- 新增 `check_background` 工具，查询单个或所有后台任务。
- 后台命令完成后通过 channel 通知主循环。
- 主循环在请求模型前注入 `<background-results>`。
- 每个后台任务写入 JSON 状态文件和 log 输出文件。
- 检测长时间 running 的 stalled task。

## Background Task 和 Task Record 的区别

s12 的 task record 是持久工作状态：

```text
这个工作要做什么、谁负责、依赖谁、状态是什么
```

s13 的 background task 是运行时执行槽位：

```text
这个 shell 命令是否还在跑、输出在哪里、是否超时
```

两者都可能落盘，但语义不同。

## BackgroundTaskRecord

```rust
pub struct BackgroundTaskRecord {
    pub id: String,
    pub status: BackgroundTaskStatus,
    pub result: Option<String>,
    pub command: String,
    pub started_at: DateTime<Utc>,
    pub finished_at: Option<DateTime<Utc>>,
    pub result_preview: String,
    pub output_file: String,
}
```

状态包括：

```rust
pub enum BackgroundTaskStatus {
    Running,
    Completed,
    Timeout,
    Error,
}
```

每个任务会产生两个文件：

```text
.runtime-tasks/<task_id>.json
.runtime-tasks/<task_id>.log
```

JSON 保存状态，log 保存完整输出。

## background_run

工具输入：

```json
{
  "command": "cargo test --workspace"
}
```

执行流程：

```text
生成 task id
  -> 写入 Running 记录
  -> tokio::spawn 后台执行 shell command
  -> 立即返回 task id
```

这让 agent 不必在当前工具调用里等待慢命令完成。

## check_background

无参数时列出所有后台任务：

```json
{}
```

传入 `task_id` 时查看单个任务详情：

```json
{
  "task_id": "19ab12cd"
}
```

单个详情会返回序列化后的 JSON view，包括本地时间展示和输出文件路径。

## 通知机制

后台命令完成后，manager 会发送：

```rust
BackgroundNotification
```

主循环在每次请求模型前调用：

```rust
self.inject_background_results();
```

如果有结果，会注入：

```xml
<background-results>
[bg:...] completed: ... (output_file=...)
</background-results>
```

这样模型会自然看到后台任务已经完成，并决定下一步动作。

## stalled task

如果任务运行超过：

```rust
pub const STALL_THRESHOLD_S: u64 = 45;
```

`background_results_message()` 会报告 stalled task id。

注意：stalled 不等于失败。真正超时由后台命令执行的 300 秒 timeout 决定。

## Rust 实现要点

- 用 `tokio::spawn` 启动后台执行。
- 用 `tokio::process::Command` 执行 shell。
- 用 `mpsc::UnboundedSender` / `UnboundedReceiver` 发送完成通知。
- 用 `AtomicU64` 生成 task id。
- 用 `Mutex<HashMap<...>>` 保存当前进程内任务状态。
- 用 UTC 时间落盘，本地时间只用于展示。

## 本章的局限

- 后台任务重启后不会自动恢复运行中的进程。
- 没有取消任务工具。
- 没有并发上限和队列策略。
- 没有权限系统保护 `background_run`。
- 任务输出只做简单截断和预览。
