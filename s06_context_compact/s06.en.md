# s06: Context Compact

As an agent works, context grows. Tool results, file contents, subagent summaries, and skill content all accumulate in the message history. Eventually the model request becomes too large or too noisy.

This chapter adds context compaction: summarize the current conversation into a compact state and continue from that summary.

## Run

```bash
cargo run -p s06_context_compact
```

## New Capabilities

- Estimate context size.
- Add a `compact` tool for manual compaction.
- Automatically compact when context exceeds a threshold.
- Summarize the conversation with the model.
- Replace long history with a compact continuation context.
- Track whether compaction has happened and preserve a recent summary.

## Why Compact

An agent context contains many things:

- User goals.
- Model reasoning.
- Tool calls.
- Tool results.
- File contents.
- Errors and retries.

Not all of this should stay forever. The model needs the current goal, key findings, decisions, modified files, and remaining work. It does not need every full tool output forever.

Compaction turns:

```text
long conversation history
```

into:

```text
short summary + current task state
```

## CompactState

The harness stores compaction metadata:

```rust
pub struct CompactState {
    pub has_compacted: bool,
    pub last_summary: Option<String>,
}
```

This lets the loop know whether history has already been compressed.

## Manual Compact Tool

The `compact` tool can be called by the model:

```json
{
  "focus": "Preserve the current implementation plan and files already edited."
}
```

The optional `focus` tells the summarizer what must be preserved.

The tool itself does not summarize immediately. It signals the main loop to compact after tool results are returned, so message ordering stays valid.

## Auto Compact

Before each model call, the loop estimates context size. If it exceeds the threshold, it triggers compaction automatically.

This is a simple heuristic, not exact tokenization. The goal is to avoid obvious runaway context growth.

## Compaction Prompt

The compaction request asks the model to preserve:

- Current goal.
- Important findings and decisions.
- Files read or changed.
- Remaining work.
- User constraints and preferences.

The output becomes a summary message that replaces the old context.

## Continuing After Compact

After compaction, the context is rebuilt into a small set of messages:

```text
system instructions
user message containing compacted summary
```

The next model call continues from the summary instead of the full raw history.

## Rust Notes

- Keep compaction as an agent runtime concern, not a normal file tool.
- Use a tool to let the model request compaction.
- Use a separate model call to summarize history.
- Replace context only after the tool result has been recorded.

## Limits

- Context size estimation is approximate.
- The summary can lose details if the prompt is not good enough.
- No transcript persistence in this chapter.
- No recent-file tracking yet.
- Compaction depends on another model call.
