# s11: Error Recovery

An agent harness should not fail immediately when a request is too large, a transport error happens, or the model output is truncated.

This chapter adds recovery paths around model calls.

## Run

```bash
cargo run -p s11_error_recovery
```

## New Capabilities

- Detect prompt-too-long errors.
- Compact context and retry.
- Detect transient transport errors.
- Retry with backoff.
- Detect max-token truncation.
- Ask the model to continue.
- Track recovery attempts to avoid infinite loops.

## Why Recovery

Long-running agents interact with unreliable systems:

- Context may grow too large.
- Network calls can fail.
- API gateways can return transient errors.
- Model output can stop at token limits.

Without recovery, the whole task fails even when the situation is fixable.

## RecoveryState

The runtime tracks recovery attempts:

- compact attempts
- transport retry attempts
- continuation attempts

This prevents endless retry loops.

## Prompt Too Long

When the model API reports that the prompt is too long, the harness:

```text
detect error
  -> compact history
  -> retry the model call
```

If compaction has already failed too many times, the error is returned.

## Transient Transport Errors

For transient failures, the harness retries with backoff:

```text
error
  -> wait
  -> retry
  -> increase attempt count
```

This helps with temporary network or gateway problems.

## Max Tokens

If the model stops because of `max_tokens`, the harness appends a continuation message:

```text
Continue from where you left off.
```

Then it calls the model again. This keeps long final answers or long reasoning outputs from being cut off abruptly.

## Rust Notes

- Keep recovery state inside runtime state.
- Use small helper functions to classify errors.
- Limit all retry paths.
- Keep compaction as the recovery mechanism for prompt-too-long errors.

## Limits

- Error classification is string-based.
- Backoff is simple.
- No persistent retry log.
- No per-provider error taxonomy.
- Continuation may still produce redundant text.
