# s12: Task System

The todo tool is useful for a single session, but it is too small for durable work tracking. A real harness needs task records that can be created, queried, updated, and linked.

This chapter introduces a task system.

## Run

```bash
cargo run -p s12_task_system
```

## New Capabilities

- Add `TaskManager`.
- Persist tasks to disk.
- Support task creation, retrieval, listing, and updates.
- Track task status.
- Track owner.
- Track dependency relationships.
- Expose task operations through separate tools.

## Why Tasks

Tasks are more durable than todos.

A task record can represent:

- A feature to implement.
- A bug to investigate.
- A refactor step.
- A background job to check later.
- A unit of work for a teammate or subagent.

Tasks become the basis for background work, worktree isolation, and multi-agent coordination.

## TaskRecord

A task record contains:

- `id`
- `subject`
- `description`
- `status`
- `blocked_by`
- `blocks`
- `owner`

Statuses include:

- `pending`
- `in_progress`
- `completed`
- `deleted`

## TaskManager

`TaskManager` handles durable task operations:

- `create`
- `get`
- `list`
- `update`
- dependency cleanup when a task is completed

Tools call the manager instead of writing task files directly.

## Task Tools

The chapter exposes separate tools:

- `task_create`
- `task_get`
- `task_list`
- `task_update`

This is clearer than one action-based `todo` tool. Each operation has its own schema and behavior.

## Dependencies

Tasks can block other tasks:

```text
task A blocks task B
```

When task A is completed, the manager removes A from other tasks' `blocked_by` lists.

This is a small but important domain rule, and it belongs in the manager, not in the tool.

## Rust Notes

- Use typed task records.
- Keep task state durable.
- Put business rules in `TaskManager`.
- Keep tools thin: parse input, call manager, render output.
- Use enums for status instead of raw strings.

## Limits

- No task priority.
- No task assignment protocol.
- No automatic scheduling.
- No claim or lease semantics yet.
- File locking is minimal.
